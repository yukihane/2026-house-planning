# 2026年 家づくり Obsidian Vault

## このファイルについて

このリポジトリーに対するエージェント向けの共通指示は、この `AGENTS.md` を正本とします。

## ディレクトリー概要

このプロジェクトは「非コードプロジェクト」です。2026年から開始した家づくりの計画、調査、ドキュメントを管理するために構成された [Obsidian](https://obsidian.md/) の Vault（保管庫）として機能します。

**ドキュメントの多くは本リポジトリー（ルートディレクトリー以下）で公開可能な形で管理されます。**一方で、個人情報などの機微情報については、公開（パブリック）リポジトリーではなく、別のプライベートリポジトリーで管理し、ローカルでは Git サブモジュール（`private-notes/`）として取り込む構成をとっています。

### 運用方針（命名規則と構造）

1. ファイル名およびディレクトリー名は、後からの検索性や一覧性を考慮し、原則として「日本語」を使用します。
2. 括弧 `()` などの記号は原則として半角を使用します。句読点 `、。` は全角を使用します。
3. 全ての Markdown ファイルには、ファイル冒頭に定型の Front Matter（YAML 形式）を記述することを必須とします。
  - `author:` フィールドには作成者を明記します。各エージェントは自分の名称を記載してください。ファイルを更新した場合も、作成者があなたでない場合は、あなたの名前を追記してください。
  - `created_at`: ファイル作成日を記載します。
  - `updated_at`: ファイル作成時にはファイル作成日を記載します。ファイルを更新した場合、更新した日付を記載します。
1. ノートを整理しやすくするため、以下の番号付きディレクトリーを標準とし、ルートディレクトリー直下に配置しています。
2. `00_受信箱/` は未整理のメモやアイデアを一旦入れる場所です。
3. `01_デイリーノート/` は日々の記録、打合せメモ、日記（日付ベースのノート、`private-notes/` 以下で管理）です。
4. `10_プロジェクト/` は現在進行形の具体的なタスク（例: 「住宅ローン」「間取り」など）です。
5. `20_領域/` は継続的に関心を持つテーマや、長期的に調査・比較検討する情報（例: 「キッチン」「外構」「震災対応・保証」「バリアフリー」など）です。
6. `30_リソース/` は参考資料、Webクリップ、カタログ情報などです。
7. `40_アーカイブ/` は完了したプロジェクトや過去の資料です。
8. `99_テンプレート/` は新規ノート作成時の雛形（テンプレートファイル）です。

## 主要なファイルとディレクトリー

* **`.obsidian/`**: Obsidian Vault のコア設定ディレクトリーです。Vault 固有の設定、テーマ、コミュニティプラグイン（`obsidian-git` や `copilot` など）が含まれています。
* **`private-notes/`**: 個人情報や機微情報など、公開できない情報を別の Git リポジトリー（プライベート）で管理するためのディレクトリーです。
* **`.gitmodules`**: 本リポジトリーの Git サブモジュールを定義しており、主に `private-notes` ディレクトリーをリンクしています。
* **`mise.toml`**: 開発ツール管理ツール `mise` の設定ファイルです。現在は Node.js バージョン 24 が指定されており、一部のプラグイン機能やローカルスクリプトの実行に必要となる場合があります。

## 使い方

1. **Obsidian で開く**: このルートディレクトリー全体を Obsidian アプリケーションの「Vault」として開いてください。
2. **サブモジュールの初期化**: 実際のノートはサブモジュール内にあるため、初期化と更新が必要です。`private-notes` ディレクトリーが空の場合は、以下のコマンドを実行してください。
   ```bash
   git submodule update --init --recursive
   ```
3. **ノートの作成**: Obsidian のインターフェースを通じて、通常のドキュメントはルートディレクトリー以下（`00_受信箱` など）に作成し、個人情報や機微情報を含むドキュメントは `private-notes/` ディレクトリー内に作成・編集してください。
4. **バージョン管理**: Obsidian Git プラグインによって Vault の自動バックアップが行われる設定になっていますが、`private-notes` ディレクトリーは独立した Git リポジトリーであるため、そのオリジンに対して個別にコミットおよびプッシュを行う必要がある点に注意してください。
